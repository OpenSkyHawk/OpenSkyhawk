37// ── controlId namespace ───────────────────────────────────────────────────────
38//
39// controlId determines how the RP2040 gateway routes an incoming packet:
40//
41// 0x0010 – 0x00FF Flight control axes & buttons (HID only — never DCS-BIOS)
42// 0x8000 – 0x86FF DCS-BIOS A-4E-C addresses (DCS-BIOS only — never HID)
43// 0xFFFF Reserved: TEST_SEQ
44//
45// Using DCS-BIOS addresses directly as controlIds in the 0x8000+ range means
46// no translation table is needed for the downstream direction (DCS → cockpit).
47// RP2040 routing logic: controlId < 0x8000 → HID, else → sendDcsBiosMessage().

57// ── Batched CAN frame (2 × ControlPacket in one 8-byte frame) ────────────────
58// MAIN_NODE should batch two ControlPackets per CAN frame when forwarding
59// DCS-BIOS state downstream. Halves bus utilisation (~30% at worst case vs ~60%

61// signals an empty/padding slot.
